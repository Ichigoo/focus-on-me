# New Module: Tasks + Home Screen

## Context

"Focus On Me" (Electron + React 19 + TS, electron-vite, Tailwind v4, SQLite via `node:sqlite`) currently has Focus, Dashboard, Adhan, and Settings sections. The user wants a new **Tasks** module: daily to-do tasks that can be scheduled daily / on specific weekdays / one-time, optionally at a specific time (with a reminder notification), markable done or ignored, with a per-task day streak and a fun, rewarding feel (confetti + chime). A new **Home screen becomes the default route `/`**, showing today's tasks plus shortcut cards to the other sections; Focus moves to `/focus`.

User-confirmed decisions:
- Home is the default landing page.
- Ignoring or missing a scheduled day **resets the streak to 0**.
- Timed tasks fire a **native notification + optional chime** (works while hidden in tray).
- Playful completion: **confetti + sound**, streak flame, "all done today" celebration.

Additional decision (default chosen, easy to revert): an **overdue one-time task carries forward** to today's Home list (labeled "overdue") until done/ignored, instead of silently disappearing after its date.

## Design decisions

- **Schedule model**: one `schedule_kind` column — `'daily' | 'weekly' | 'once'`. "Repeat" in the UI is just the toggle between `once` (date picker) and `daily`/`weekly`. No invalid combos.
- **Weekdays**: integer bitmask, `bit = 1 << Date.getDay()` (bit 0 = Sunday). UI renders Monday-first visually only. Keep this convention consistent across repo/scheduler/UI — easiest bug to introduce.
- **Done/ignored**: `task_completions(task_id, day, status)` with composite PK, one row per task per local day (`YYYY-MM-DD`, same convention as `localDayString()` in `src/main/db/stats.ts:18`). Upsert to set, delete to undo.
- **Streak**: computed **lazily in main-process JS** at query time from completions + schedule (no backfill rows, no midnight write job) — same read-time approach as the existing focus streak in `stats.ts:42-58`. A gap in completions *is* the reset.
- **Reminders**: new `src/main/tasks/scheduler.ts` cloning `src/main/adhan/scheduler.ts` (setTimeout array, midnight re-arm, re-arm on data/settings change).
- **Sounds**: two new `SoundKind`s — `'task-reminder'`, `'task-done'` — generated by extending `scripts/gen-sounds.mjs`. Completion chime plays directly in the renderer (main window owns audio) for instant feedback.
- **Confetti**: no npm dep — small `ConfettiBurst` component with the Web Animations API; respects `prefers-reduced-motion`.
- **Two pages**: `Home.tsx` at `/` (glanceable today view + shortcuts) and `Tasks.tsx` at `/tasks` (CRUD management), so the Home stays clean.

## Step 1 — Migration (`src/main/db/index.ts`)

Append one string to `MIGRATIONS` (becomes v2; `migrate()` handles the rest):

```sql
CREATE TABLE tasks (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  name TEXT NOT NULL,
  schedule_kind TEXT NOT NULL DEFAULT 'daily',   -- 'daily' | 'weekly' | 'once'
  weekdays INTEGER NOT NULL DEFAULT 127,          -- bitmask, 1<<getDay() (bit0=Sun); used when kind='weekly'
  once_date TEXT,                                 -- local 'YYYY-MM-DD'; used when kind='once'
  time_hhmm TEXT,                                 -- local 'HH:MM' or NULL = anytime
  archived INTEGER NOT NULL DEFAULT 0,
  created_at INTEGER NOT NULL,                    -- epoch seconds
  created_day TEXT NOT NULL                       -- streak scan never goes before this
);
CREATE TABLE task_completions (
  task_id INTEGER NOT NULL REFERENCES tasks(id) ON DELETE CASCADE,
  day TEXT NOT NULL,
  status TEXT NOT NULL,                           -- 'done' | 'ignored'
  marked_at INTEGER NOT NULL,
  PRIMARY KEY (task_id, day)
);
CREATE INDEX idx_task_completions_day ON task_completions(day);
```

`PRAGMA foreign_keys = ON` already set (index.ts:110) so cascade works. No seed changes.

## Step 2 — Shared types (`src/shared/types.ts`)

```ts
export type TaskScheduleKind = 'daily' | 'weekly' | 'once'
export type TaskDayStatus = 'done' | 'ignored'

export interface Task {
  id: number
  name: string
  schedule_kind: TaskScheduleKind
  weekdays: number
  once_date: string | null
  time_hhmm: string | null
  archived: number
  created_at: number
  created_day: string
}
export type TaskInput = Pick<Task, 'name' | 'schedule_kind' | 'weekdays' | 'once_date' | 'time_hhmm'>
export interface TaskWithStatus extends Task {
  status: TaskDayStatus | 'pending'
  streak: number
  overdue?: boolean   // carried-forward 'once' task past its date
}
```

- Extend `SoundKind` (types.ts:100): `| 'task-reminder' | 'task-done'`.
- Extend `AppSettings` + `DEFAULT_SETTINGS`: `taskRemindersEnabled: boolean` (true), `soundTaskReminder: boolean` (true).
- Extend `Api` with a `tasks` namespace: `list`, `create`, `update`, `remove`, `listForDay(day)`, `setStatus(id, day, status | null)` (returns fresh day list to avoid a second round-trip), `onChanged` (subscribes to `tasks:changed` broadcast).

## Step 3 — Repo (`src/main/db/tasks.ts`, new file)

Export `localDayString` from `stats.ts` (currently private) and reuse it.

- `isScheduledOn(task, day)`: `once` → `once_date === day` (plus carry-forward: for the *today* query, an unmarked `once` task with `once_date < day` is included with `overdue: true`); `weekly` → `weekdays & (1 << new Date(day + 'T00:00:00').getDay())`; `daily` → true. Never before `created_day`.
- Streak (per task): load its completions into a `Map<day, status>`, walk back from the queried day: today unmarked → skip without breaking (same tolerance as stats.ts:52); unscheduled day → continue; scheduled + done → streak++; scheduled + ignored or missing → stop; stop below `created_day`; 3650-day safety cap.
- `tasksRepo`: `list()`, `get(id)`, `create(input)` (sets `created_day`), `update(id, input)`, `remove(id)`, `setStatus(taskId, day, status | null)` (`ON CONFLICT(task_id, day) DO UPDATE`, or DELETE when null), `listForDay(day)` (filter in JS, join status + streak, sort by `time_hhmm` nulls-last then name).
- Validation: reject `weekly` with `weekdays === 0` and `once` with null `once_date` (also prevented in UI).

## Step 5 — Reminder scheduler (`src/main/tasks/scheduler.ts`, new file)

Clone of `adhan/scheduler.ts`:
- `rescheduleTasks()`: clear timers; if `taskRemindersEnabled`, arm a setTimeout for each of today's tasks with `time_hhmm` still `pending` and in the future; re-arm shortly after local midnight (`setHours(24, 0, 5, 0)`) and **also `broadcast('tasks:changed')` at midnight** so Home rolls to the new day.
- `fireTask(id)`: re-fetch and re-check status is still `pending` (correctness independent of reschedule timing); native `Notification` ("Task reminder" / task name); `broadcast('sound:play', 'task-reminder')` if `soundTaskReminder`.
- Wire `rescheduleTasks()` in `src/main/index.ts` right after `rescheduleAdhan()` (index.ts:38).

## Risks

- **Weekday bitmask consistency** between repo, scheduler, and Mon-first UI — the likeliest bug.
- **DST**: local-day + midnight re-arm matches the accepted adhan/stats tolerance (a timed task may fire an hour off on a DST-jump day).
- **Route move**: main window now opens on Home in all show paths (`session-ended`, tray, widget) — the session banner mitigates.
